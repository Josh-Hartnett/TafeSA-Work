Tafe SA Learn & Web Content Accessibility Guidelines (WCAG)

The International Web Content Accessibility Guidelines explain how to make web content more accessible to people with disabilities. Web "content" generally refers to the information in a web page or web application. Some common guidelines from WCAG that TAFE SA online learning content needs to follow can be found by hovering over the images below. The links within the images below provide details on meeting web content accessibility standards


Non-text content that is presented to the user has a text alternaive

If non-text content is time-based media (video or audio), then text alternaives at least provide descriptive identification of the non-text content.

Color is not used as the only visual means of conveying information

All functionality of the content is operable through a keyboard interface

Tafe SA E-standards for Content Formats

Tafe SA E-standards have been guided by former national VET E-Learning Strategy guidelines. These standards included recommended content formats and file types. The main aims of these content format standards are to ensure sustainability and accessibility. The main file format types acceptable for us in the Tafe SA Learn learning environment are:

Digital Item File Format Common software used to develop
Images jpeg, jpg MS Paint, Photoshop
Video & Animation mp4, html5 MovAvi, Skype for Business - Recording, Office Mix, MS MovieMaker, Captivate 9 or later
NOTE: Flash files should be replaced with html5 format.
Audio mp3 Audacity
Text pdf files, doc, docx files MS Word, Acrobat DC, Acrobat Pro
Presentation ppt, pptx files MS Powerpoint
Packaged learning files SCORM 1.2 Captivate 9 or later
